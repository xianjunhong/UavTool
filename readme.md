# UavTool

UavTool 是一个面向无人机航点作业的桌面工具，核心目标是：

1. 在 GeoTIFF 上快速选点并生成经纬度航点。
2. 将航点按 DJI WPML 格式导出为 KMZ。
3. 对无金字塔的 TIF 进行 overviews 构建，提升大图缩放性能。
4. 对两幅影像进行控制点配准，并输出新的对齐 TIF。
5. 图像裁剪支持自相交点序自动修复与透明边输出。
6. 小区裁剪支持小区库复用与批量导出（TIF/PNG）。

## 当前可以做什么

### 1. 航线绘制页

1. 导入 TIF 并自动检查是否包含金字塔（无金字塔时拒绝导入）。
2. 大图浏览：
   - 鼠标滚轮缩放。
   - 左键按住拖动平移。
   - 右键按住也可平移。
3. 航点编辑：
   - 左键点击添加航点。
   - 右键点击航点附近可删除航点。
   - 图上显示编号航点，右侧同步显示经纬度列表。
   - 列表每行有 x 删除按钮，可删除对应航点。
   - 双击列表项可定位到图上对应点。
4. 导出 KMZ：
   - 点击“导出航线”。
   - 在参数弹窗中设置无人机型号、速度、高度、偏航模式、云台角度、输出路径。
   - 生成包含 wpmz/template.kml 和 wpmz/waylines.wpml 的 .kmz 文件。

### 2. 金字塔构建页

1. 选择 TIF 后自动检测金字塔层级。
2. 若已包含金字塔，直接提示并禁用构建按钮。
3. 若不包含金字塔，可一键构建 overviews。
4. 构建过程异步执行并显示进度条，不阻塞 UI。

### 3. 图像配准页

1. 选择 src 图像和 target 图像。
2. 在左图按顺序打 src 控制点，在右图按同顺序打 target 控制点。
3. 至少需要 3 对点，建议 4 对及以上，且尽量分散分布。
4. 点击“图像对齐”后异步执行并显示进度条。
5. 配准结果输出为新的 TIF 文件（默认 *_aligned.tif），不会修改原 target 文件。
6. 对齐输出会自动构建金字塔，便于后续快速浏览。
7. 完成后会显示误差与稳定性指标：
   - 训练 RMSE（控制点拟合误差）
   - 最大误差
   - 留一 RMSE（点数 >= 4 时）
   - det(A) 与 cond(A)（用于判断变换是否退化）

### 4. 图像裁剪页

1. 选择待裁剪影像后，在图上左键绘制多边形，右键撤销最后一个点。
2. 支持两种输出模式：
   - 输出新图（默认 *_crop.tif）
   - 覆盖原图
3. 支持自动修复沙漏型（自相交）点序，避免因点序导致裁剪失败。
4. 裁剪输出带 Alpha 掩膜，查看器按透明显示，减少黑边视觉干扰。
5. 裁剪过程异步执行并显示进度；GeoTIFF 输出完成后自动构建金字塔。

### 5. 小区裁剪页

1. 左侧影像交互区 + 右侧小区列表，支持小区绘制与管理。
2. 小区库管理：
   - 添加当前小区（命名）
   - 编辑/重命名选中小区
   - 删除小区
   - 保存小区库（Shapefile）
   - 加载小区库（支持 shp/geojson/gpkg）
3. 小区显示与应用：
   - 已保存小区在图上显示轮廓与名称。
   - 可将选中小区应用到当前编辑多边形并自动定位。
4. 批量导出：
   - 选择输出文件夹后按小区名称逐个导出。
   - 默认导出 GeoTIFF。
   - 勾选“裁剪导出为 PNG”后批量导出 PNG 原图。

## 页面结构

应用使用 QStackedWidget 五页结构，并通过顶部 Tab 栏切换：

1. 航线绘制
2. 金字塔构建
3. 图像配准
4. 图像裁剪
5. 小区裁剪

## 技术栈

1. PySide6: 桌面 UI
2. GDAL: GeoTIFF 读取、坐标写回与金字塔构建
3. pyproj: 坐标系转换（影像投影 -> WGS84）
4. XML + zipfile: WPML 生成与 KMZ 打包
5. NumPy: 仿射参数求解与配准误差计算

## 目录说明

1. main.py: 程序入口
2. ui/: 页面与图像交互组件
3. logic/: KMZ 导出、金字塔构建、图像配准、图像裁剪、小区库读写与批量导出逻辑
4. utils/: 运行环境与地理工具

## 已知说明

1. 航线绘制页仅支持 RGB 三波段及以上影像。
2. KMZ 导出基于当前航点顺序，按点位附加拍照动作。
3. 无人机型号差异目前主要体现在 WPML 中的机型/载荷枚举参数。
4. 配准时若控制点近共线，矩阵可能退化（det(A) 过小或 cond(A) 过大），系统会拒绝写出并提示重选点。
5. 当仅使用 3 对点时，训练 RMSE 可能接近 0，这是仿射模型精确拟合控制点导致，不代表全图误差为 0。
6. 小区库保存使用影像坐标系，加载时会尝试自动转换到当前影像坐标系。
7. PNG 输出不包含 GeoTIFF 地理参考信息；若需保留地理坐标，请使用 TIF 导出。
